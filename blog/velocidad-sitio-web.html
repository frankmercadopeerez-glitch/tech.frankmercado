En 2026, la velocidad de tu sitio web no es un lujo técnico - es un factor crítico de negocio. Google la usa para rankear tu sitio, y tus clientes la usan para decidir si quedarse o irse a la competencia.

¿Qué son los Core Web Vitals?

Google mide la experiencia de usuario con tres métricas principales que ahora son factores de ranking:

  • LCP (Largest Contentful Paint): Tiempo hasta que el contenido principal es visible. Ideal: menos de 2.5 segundos.
  • INP (Interaction to Next Paint): Qué tan rápido responde la página a interacciones. Ideal: menos de 200ms.
  • CLS (Cumulative Layout Shift): Estabilidad visual - que los elementos no "salten" mientras carga. Ideal: menos de 0.1.

El Impacto Real en tu Negocio

Estos no son números abstractos. Walmart descubrió que por cada segundo de mejora en velocidad, sus conversiones aumentaban un 2%. Amazon calculó que un retraso de 100ms les costaba 1% en ventas.

Para un negocio local en Cartagena, un sitio lento significa:

  • Clientes que se van a la competencia (que probablemente tiene mejor sitio)
  • Menor posicionamiento en búsquedas locales
  • Mayor costo por clic en Google Ads (Google penaliza páginas lentas)
  • Peor experiencia en móviles con conexiones 3G/4G

Cómo Diagnosticar tu Velocidad

Usa estas herramientas gratuitas para ver cómo está tu sitio:

  • PageSpeed Insights: pagespeed.web.dev - El diagnóstico oficial de Google
  • GTmetrix: Análisis detallado con waterfall de carga
  • WebPageTest: Test desde diferentes ubicaciones y conexiones

7 Optimizaciones que Puedes Hacer Hoy

1. Optimiza Imágenes

Las imágenes son típicamente el 50-80% del peso de una página. Usa formatos WebP, comprime, y carga lazy loading.

2. Minimiza CSS y JavaScript

Elimina código no usado y combina archivos para reducir peticiones HTTP.

3. Usa un CDN

Cloudflare (gratuito) sirve tu contenido desde servidores cercanos a tus usuarios.

4. Habilita Compresión Gzip

Reduce el tamaño de archivos de texto en 70-90%.

5. Cachea Agresivamente

Configura headers de caché para que los navegadores guarden recursos estáticos.

6. Preconecta a Recursos Externos

Usa rel="preconnect" para Google Fonts y otros recursos de terceros.

7. Elimina Render-Blocking Resources

Carga CSS crítico inline y difiere JavaScript no esencial.

¿Cuándo es "Suficientemente Rápido"?

Para la mayoría de negocios, estos son buenos objetivos:

  • Tiempo de carga: Menos de 3 segundos en 3G
  • PageSpeed Score: 80+ en móvil, 90+ en desktop
  • Peso total: Menos de 3MB para la página inicial

Conclusión

La velocidad de tu sitio web es una inversión con retorno medible. Cada mejora se traduce directamente en mejor SEO, más conversiones, y clientes más satisfechos.
